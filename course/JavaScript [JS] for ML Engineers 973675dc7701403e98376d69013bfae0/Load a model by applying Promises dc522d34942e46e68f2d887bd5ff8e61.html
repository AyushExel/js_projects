Load a model by applying Promises

Now it’s time to use a library called magenta.js. It allows you to use artistic Deep neural net-based models in the browser. Let’s load the default magenta model used for style transfer.

As you might have already notied, App.js imports magenta’s image library as the top import * as mi from "@magenta/image";

Let’s start with creating a model. 

const [model, setModel] = useState(new mi.ArbitraryStyleTransferNetwork());

This ArbitraryStyleTransferNetwork loads the deafult style transfer model from magenta image.

Note

For this project, we don’t support multiple models so using the useState hook isn't required but it’s a fun adventure that you can take up after so I’m leaving it as it is.

Loading the model on start-up

We want to load the initialize the loaded model as soon as the page loads. We’ll use the useEffect hook for this.

useEffect(() => {
    model.initialize()
  }, 
[model]);

Here, we’re passing model as the second argument, which means that this effect will only be applied if model changes. This means that we’ll initialize every time a new model is loaded and we won’t re-initialize the same model.

initialize() returns a promise

initialize() is an async operation that returns a Promise. Ideally, we want to want to wait for the model to get initialized before we let the user interact with the app.

So let’s return a spinner while the model gets initialized in the background.

Displaying a spinner

There are many ways to display a loader screen. We’ll use a very simple logic using useState hook for conditional rendering.

const [initialized, SetInitialized] = useState(false);

initialized will be set to false by default and we’ll update it to true as soon as the model is initialized.

Let’s update the effect:

useEffect(() => {
    model
      .initialize()
      .then(() => {
        SetInitialized(true);
      })
      .catch((error) => console.log(error));
  }, [model]);

conditional rendering

Add this right before the original return statement

if (!initialized) {
    return <div className="loader"></div>;
  }

the boilerplate css will render this div with class loader as a spinner. Once the model is initialized, the page will re-render as the state of initialized changes

Here’s how app.js will look like at this stage:

    function App({ defaultContent, defaultStyle }) {
  const [content, setContent] = useState(defaultContent);
  const [style, setStyle] = useState(defaultStyle);
  const [initialized, SetInitialized] = useState(false);
  const [strength, setStrength] = useState(0.5);
  const [model, setModel] = useState(new mi.ArbitraryStyleTransferNetwork());
  const [result, setResult] = useState("");

  const handleContentImageChange = (e) => {
    setContent(URL.createObjectURL(e.target.files[0]));
  };

  useEffect(() => {
    model
      .initialize()
      .then(() => {
        SetInitialized(true);
      })
      .catch((error) => console.log(error));
  }, [model]);

  if (!initialized) {
    return <div className="loader"></div>;
  }

  return (
    <div className="App">
      <div className="row">
        <div className="column">
          <img src={content} alt="content" />
          <br />
          <input type="file" onChange={handleContentImageChange} />
        </div>

        <div className="column">
          <img src={style} alt="style" />
          <br />
          <input
            type="file"
            onChange={(e) => setStyle(URL.createObjectURL(e.target.files[0]))}
          />
        </div>

        <div className="column">
          <img src={result} />
        </div>
      </div>
      <div className="row slide_container">
        <input
          type="range"
          min="0.0"
          max="100.0"
          className="slider"
          onChange={(e) => {
            setStrength(e.target.value / 100);
          }}
        />
      </div>
    </div>
  );
}
